## 2. 数据准备

### 2.1 数据来源
直接读取 `Task-3/兆易创新_daily.csv`，已包含：
- `trade_date, open, high, low, close, vol, amount`

### 2.2 必要字段衍生
在策略计算过程中新增以下列：
- `true_range`: 当日 True Range = max(high - low, |high - prev_close|, |low - prev_close|)
- `n_value`: N = (前19日 N × 19 + 当日 TR × 2) / 21 （指数平滑）
- `channel_high_20`: 过去20日最高点（S1 入场上轨）
- `channel_low_10`: 过去10日最低点（S1 退出下轨）
- `channel_high_55`: 过去55日最高点（S2 入场上轨）
- `channel_low_20`: 过去20日最低点（S2 退出下轨）

## 2.3 头寸计算（Position Sizing）

海龟策略的核心风控：**每单位头寸的大小 = 账户总资金的 1% ÷ N 值**

```
1 单位股数 = floor( (账户总资金 × risk_pct) / N / 100 ) × 100
```

其中 `risk_pct = 0.01`（1%）。这样每单位的最大理论亏损被锁定在账户的 1%：
- 止损宽度 = 2N
- 每单位最大亏损 = 2N × 单位股数 = 2 × (账户 × 1% ÷ N) × N = 账户 × 2%

**示例**（以 N=2.0 元、账户 100 万为例）：
```
1 单位股数 = floor( (1,000,000 × 0.01) / 2.0 / 100 ) × 100
           = floor( 50 ) × 100
           = 5,000 股
```

头寸规模随 N 值**动态调整**：
- 波动率升高（N 变大）→ 自动减仓
- 波动率降低（N 变小）→ 自动加仓

## 2.4 止盈机制（趋势退出即止盈）

海龟策略**没有百分比止盈**——它与传统策略的关键区别在于 **"让利润奔跑"**：

| 系统 | 止盈条件 | 本质 |
|------|---------|------|
| S1 | 收盘价跌破过去 **10日**最低点 | 趋势反转确认后离场 |
| S2 | 收盘价跌破过去 **20日**最低点 | 趋势反转确认后离场 |

- 退出 = 止盈（止损止盈一体化）：通道下轨既是止盈线也是反向止损线
- 这本质上是一种**移动止盈**：随着趋势上行，通道下轨也在逐步抬高
- 代价是利润回吐较大（可能从高点回撤 20-30%），但这样才能吃到整段趋势

## 3. 策略设计 (turtle_strategy.py)

### 3.1 类结构

```
TurtleStrategy(BaseStrategy)
├── __init__(self, system="S1", entry_window=20, exit_window=10,
│             atr_period=20, risk_pct=0.01, max_units=4, add_unit_step=0.5)
│
├── calc_true_range(df)          -> pd.Series  # 计算 TR
├── calc_n_value(df)             -> pd.Series  # 指数平滑 N
├── calc_donchian_channel(df, window) -> pd.Series  # 计算通道上下轨
├── generate_signals(df)         -> pd.DataFrame
│   ├── 计算 TR → N → 通道
│   ├── 入场信号: 当日最高价突破通道上轨 (high > channel_high) → signal=1
│   ├── 加仓信号: 持仓中且价格较上次买入涨了 +0.5N → signal=2
│   ├── 止损信号: 收盘价跌破各单位止损价 → signal=-2
│   └── 退出信号: 当日最低价跌破通道下轨 (low < channel_low) → signal=-1
│
└── __repr__()
```

### 3.2 信号编码
| signal 值 | 含义 |
|-----------|------|
| 0 | 无操作 |
| 1 | 首次入场买入（开第一单位） |
| 2 | 金字塔加仓（开第2~4单位） |
| -1 | 趋势退出（通道下轨跌破，全部平仓） |
| -2 | 止损触发（2N 止损，触发单位平仓） |

### 3.3 参数默认值
| 参数 | S1 默认值 | S2 默认值 |
|------|----------|----------|
| entry_window | 20 | 55 |
| exit_window | 10 | 20 |
| atr_period | 20 | 20 |
| risk_pct | 0.01 (1%) | 0.01 |
| max_units | 4 | 4 |
| add_unit_step | 0.5 (0.5N) | 0.5 |

## 4. 回测引擎设计 (turtle_backtest.py)

### 4.1 与 Task-3 回测引擎的关键差异

```
Task-3 BacktestEngine:          海龟 TurtleBacktestEngine:
┌──────────────────────┐        ┌──────────────────────────┐
│ 单一持仓 (shares)     │        │ 多单位持仓台账           │
│ 全仓进出              │        │ 逐单位管理               │
│ signal ∈ {0, 1, -1}  │        │ signal ∈ {0,1,2,-1,-2}  │
│ 简单止损/止盈         │        │ 2N固定止损 + 通道退出    │
└──────────────────────┘        └──────────────────────────┘
```

### 4.2 持仓台账结构

```python
position_units = [
    {
        "unit_id": 1,
        "entry_date": "2024-03-15",
        "entry_price": 85.50,
        "stop_price": 81.10,    # entry - 2N
        "shares": 1200,         # 按N值计算的股数
        "active": True,         # 是否仍在持仓
    },
    # unit 2, 3, 4 ...
]
```

### 4.3 逐日处理逻辑

```
遍历每一天的行情数据:
│
├── 1. 检查已有仓位是否触发止损
│   └── 某单位的 close <= stop_price → 平掉该单位 (signal=-2)
│
├── 2. 检查是否触发趋势退出
│   └── 所有单位的 close <= channel_low → 全部平仓 (signal=-1)
│
├── 3. 检查是否满足入场条件
│   ├── 当前无持仓 且 close > channel_high → 开第一单位 (signal=1)
│   └── 计算该单位的 shares = (capital × risk_pct) / N / 100 × 100
│
├── 4. 检查是否满足加仓条件
│   ├── 已有持仓 < max_units
│   └── 当前最高价 >= 上一单位 entry_price + add_unit_step × N → 加一单位 (signal=2)
│
└── 5. 记录当日权益
    └── equity = cash + Σ(active_unit.shares × close)
```

### 4.4 交易成本

- 手续费率: 0.0003（双边，最低5元）
- 滑点: 0.0001
- 印花税: 0.001（仅卖出单边）

## 7. 绩效指标

| 指标 | 计算方式 |
|------|---------|
| 总收益率 | (最终权益 - 初始资金) / 初始资金 × 100% |
| 年化收益率 | (1 + 总收益率) ^ (252/回测天数) - 1 |
| 最大回撤 | min((权益 - 峰值) / 峰值) × 100% |
| 夏普比率 | (日超额收益均值 / 日超额收益标准差) × √252 |
| 胜率 | 盈利交易数 / 总交易数 × 100% |
| 盈亏比 | 平均盈利 / 平均亏损（绝对值） |
| 盈利因子 | 总盈利 / 总亏损（绝对值） |
| 交易次数 | 总交易笔数 |
| 平均持仓天数 | 所有交易的 holding_days 均值 |

## 9. 注意事项与边界情况

1. **N值初始化**: 前20天没有足够的 TR 数据来计算可靠的 N 值，这些天不做交易
2. **数据缺失**: 使用 `min_periods` 参数，不足时返回 NaN
3. **通道冷启动**: 前 entry_window 天没有通道数据，不做交易
4. **整数手数**: A股买股必须为 100 股的整数倍（手），用 `int(shares / 100) * 100`
5. **最低佣金**: 佣金不足 5 元按 5 元收取
6. **资金不足加仓**: 若现金不足以买入一个最小单位（100股），则跳过本次加仓
7. **S1 / S2 冲突**: 两个系统不互斥，但回测时建议分别运行再对比
8. **最后一日平仓**: 若回测结束日仍持仓，按收盘价强制平仓
